基于 GTD + 四象限法则的待办管理系统（独立子用户）

**用户**: wife
**创建日期**: 2026-03-07
**状态**: ✅ 已激活（v0.2.1 数据层已初始化）

---

## 📁 目录结构

```
users/wife/
├── data/
│   └── tasks.json    # wife 的结构化主库（唯一事实源）
├── inbox.md          # 收集箱/总览（由主库生成）
├── today.md          # 今日待办（由主库生成）
├── matrix/           # 四象限视图（由主库生成）
├── archive/          # 已完成归档
├── scripts/          # wife 专用脚本预留目录
├── morning-brief.sh  # 早间提醒脚本
└── daily-reminder.sh # 晚间提醒脚本
```

---

## 🔄 使用流程

1. **收集/更新** → 新任务直接进入 wife 的结构化主库
2. **主库存储** → 统一写入 `users/wife/data/tasks.json`
3. **展示渲染** → `today.md / inbox.md / matrix/*` 作为展示层
4. **提醒** → 早 10 点、晚 8:30 读取展示层发送提醒
5. **执行/完成** → 完成后从待办视图移除，进入已处理
