# GTD × Apple Reminders 重构交付说明 v1.0

## 本次完成内容

### 1. 状态语义收口
- done 与 delete 彻底分离
- delete 采用 `deleted_at` 真删除语义
- reopen 可恢复 open 状态

### 2. 分类命名统一
- category 从 `index` 统一改为 `inbox`
- 代码、数据、Apple mapping、文档同步更新

### 3. 视图规则收口
- 主清单只显示 `open && 未 deleted`
- done/cancelled/archived 进入完成区
- deleted 不进入常规视图

### 4. CLI 能力
- 新增真正的 `delete` 命令
- `list` 默认不显示 deleted
- `list --include-deleted` 可查看删除项

### 5. Apple 单向同步第一轮收口
- export summary 更清晰
- sync state 第一轮精简
- 保持 GTD -> Apple 主链不扩功能

### 6. Apple completed 回传
- 新增最小回传链路：`completed -> done`
- 事件必须携带 `gtd_id`
- 按 `event_id` 幂等去重
- 重复导入无副作用
