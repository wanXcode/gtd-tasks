# GTD 时间规则

## 固定时区

GTD 系统统一使用：**Asia/Shanghai（UTC+8，北京时间）**

这条规则适用于：
- `scripts/gtd_reminder_digest.py` 的 morning / evening 模式
- today.md 的“今日”语义
- 手动查看“待办清单”时的分类
- 早上 10 点提醒
- 晚上 8:30 提醒
- “今天 / 明天 / 昨日 / 下周 / 周几 / 日期” 的全部判断

## 重要说明

- 服务器运行时区可以是 UTC
- 但 GTD 的业务时间口径 **不能** 跟着服务器 UTC 走
- 只要是 GTD 相关输出，一律按北京时间判断

## 分类规则

1. `today.md` 中的未完成事项，默认归到 **【今日】**
2. Q1 中未完成事项，如果没有更明确时间信息，优先归到 **【明日】** 或近期推进
3. Q2/Q3/Q4 中未完成事项，没有明确近期时间时，归到 **【未来】**
4. 任务文本里如果出现“明天 / 明日 / 下周 / 周几 / 日期”，优先按文本时间理解，但解释基准仍然是北京时间

## 目的

避免因为服务器是 UTC，导致北京时间已跨天后，仍把 GTD 的“今日”错误判空。
