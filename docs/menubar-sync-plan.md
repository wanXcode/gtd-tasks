# GTD Menubar Sync 产品方案

## 背景

当前 GTD → Apple Reminders 的 Mac 同步链路依赖 `launchd + Python + reminders-bridge`。已验证：

- 在当前交互/网关上下文中，`reminders-bridge preflight` 返回 `authorized`
- 在 `launchd` 后台上下文中，同一个 bridge 返回 `not_determined`

这意味着问题不在同步业务逻辑本身，而在 **后台权限主体不稳定**。因此决定将 Reminders / EventKit 的访问权收敛到 **原生 Swift menubar app**。

## 产品目标

构建一个登录后常驻的 macOS 菜单栏应用，作为：

1. GTD → Apple Reminders 同步的唯一 EventKit 权限主体
2. 本地同步状态面板
3. 手动触发与问题可视化入口

## 非目标

MVP 不做：

- 完整 GTD 编辑界面
- 多账户切换
- 复杂设置页
- 双向复杂冲突解决 UI
- 推送/WebSocket 实时同步
- 脱离现有服务端 API 重写同步协议

## 目标用户

- 当前这台 Mac 的主使用者
- 已经在使用 GTD 服务端、Apple Reminders、现有同步规则

## 核心用户价值

### 1. 稳定
不再依赖 launchd 后台上下文访问 EventKit。

### 2. 可见
用户能在菜单栏看到：

- 权限状态
- 服务端状态
- 上次同步时间
- 最近错误
- 当前同步状态

### 3. 可控
用户可以：

- 立即同步
- 请求 Reminders 权限
- 查看错误摘要
- 必要时暂停自动同步

## 核心用户流程

### 流程 A：首次启动
1. 用户登录 Mac
2. menubar app 自动启动
3. app 检查 Reminders 权限
4. 若未授权，显示状态并提供“请求权限”
5. 授权成功后，进入自动同步

### 流程 B：自动同步
1. 定时器触发
2. 检查权限与服务端连通性
3. 拉取 changes
4. 写入 / 更新 / 完成 Reminders
5. 成功后 ack
6. 更新本地状态与菜单栏显示

### 流程 C：同步失败
1. 单条任务失败时记录错误
2. 菜单栏显示最近错误摘要
3. 不影响其他任务继续同步
4. 不对失败变更做错误 ack

### 流程 D：权限异常
1. app 检测到未授权/受限
2. 自动停止 EventKit 写入
3. 菜单栏状态切换为异常
4. 用户可手动请求权限或根据指引恢复

## MVP 范围

### 必须有
- 菜单栏常驻图标
- 启动自动检查权限
- 手动“立即同步”
- 自动定时同步（默认 60 秒）
- 显示权限状态
- 显示服务端状态
- 显示上次同步时间
- 显示最近错误摘要
- 调用现有 changes / tasks / mappings / ack API
- 支持 reminder create / update / move / complete / delete
- 本地保存 cursor、mapping、最近同步状态

### 可延后
- completed 反向回写
- 高级设置面板
- 日历选择 UI
- 一键诊断详情页
- 自动自愈历史脏 mapping

## 菜单栏状态设计

### 顶层状态
- `正常同步`
- `同步中`
- `权限未授权`
- `服务端异常`
- `部分失败`

### 菜单内容建议
- 同步状态：正常 / 同步中 / 异常
- Reminders 权限：已授权 / 未授权 / 受限
- 服务端：在线 / 离线
- 上次成功同步：时间戳
- 最近错误：一行摘要
- 按钮：立即同步
- 按钮：请求权限
- 按钮：打开日志目录 / 打开数据目录（可选）
- 按钮：退出

## 失败处理原则

1. **权限失败优先暴露，不静默吞掉**
2. **单条失败不拖垮整轮同步**
3. **只 ack 连续成功消费的 change**
4. **状态必须显式可见**
5. **禁止后台旧链路与新 app 同时写入 Reminders**

## 迁移策略

### 阶段 1：并行开发
- 新建 menubar app
- 沿用现有服务端 API
- 沿用现有同步规则和映射逻辑

### 阶段 2：受控试运行
- 在本机测试环境启动新 app
- 暂停旧 launchd 的 EventKit 主写入通路
- 验证权限稳定性、状态展示、基本同步正确性

### 阶段 3：主通路切换
- 以 menubar app 作为唯一 EventKit 主体
- 旧 Python 同步器降级为参考实现或下线

## 成功标准

### MVP 成功标准
- menubar app 在前台上下文下能稳定请求并保有 Reminders 权限
- 可以稳定完成一轮增量同步
- 能显示健康状态与最近错误
- 同步不会因 launchd 上下文权限异常而停摆

### 切换成功标准
- 关闭旧 launchd 写入后，新 app 能独立完成日常同步
- 不重复创建 reminders
- 不错误 ack 未成功的 changes
